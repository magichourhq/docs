Magic Hour surfaces failures in two places: **HTTP status codes** on the API request itself, and **application error codes** on a job that fails while rendering. Handle both to build a robust integration.

## HTTP status codes

Returned on the response to your API request. Error responses have a JSON body with a `message` field:

```json
{ "message": "Unauthorized" }
```

| Status | Meaning               | Common cause                                          | What to do                                                                 |
| :----- | :-------------------- | :---------------------------------------------------- | :------------------------------------------------------------------------- |
| `400`  | Bad Request           | Missing or malformed request body / parameters        | Check your JSON body and parameter values against the endpoint reference   |
| `401`  | Unauthorized          | Missing, invalid, or revoked API key                  | Verify the `Authorization: Bearer <key>` header and that the key is active |
| `402`  | Payment Required      | Not enough credits to start the job                   | [Add credits](https://magichour.ai/dashboard/my-plan) or upgrade your plan |
| `404`  | Not Found             | The requested resource (e.g. project `id`) is unknown | Confirm the `id` and that the project still exists                         |
| `422`  | Unprocessable Entity  | The request is valid but cannot be processed          | Review the `message` for the specific reason and adjust your inputs        |
| `500`  | Internal Server Error | Temporary server issue                                | Retry after a few seconds                                                  |
| `502`  | Bad Gateway           | Server temporarily unavailable                        | Retry after 30–60 seconds                                                  |
| `503`  | Service Unavailable   | Server overloaded                                     | Retry with exponential backoff                                             |

## Application error codes

When a request is accepted (HTTP `2xx`) but the job fails during rendering, the project's `status` becomes `error` and the `error` object contains a machine-readable `code` and a human-readable `message`:

```json
{
  "status": "error",
  "error": {
    "code": "no_source_face",
    "message": "Please use an image with a detectable face"
  }
}
```

This is the same payload delivered to your `video.errored`, `image.errored`, and `audio.errored` [webhook events](/integration/webhook/event-types).

| Code                       | Seen on      | Cause                                       | Remedy                                                                                        |
| :------------------------- | :----------- | :------------------------------------------ | :-------------------------------------------------------------------------------------------- |
| `invalid_video_file`       | Video        | Corrupted or unsupported video              | Re-encode or use a [supported format](/integration/inputs-and-outputs#supported-file-formats) |
| `invalid_file_format`      | Image        | Unsupported image format                    | Use a [supported image format](/integration/inputs-and-outputs#supported-file-formats)        |
| `file_too_large`           | Video, Image | File exceeds your tier's upload limit       | Compress the file or upgrade your tier                                                        |
| `no_source_face`           | Image, Video | No detectable face in the provided input    | Use an input that contains a clear, detectable face                                           |
| `content_policy_violation` | Image        | Input or prompt violates content guidelines | Adjust the input or prompt                                                                    |
| `text_too_long`            | Audio        | Input text exceeds the maximum length       | Shorten the input text                                                                        |
| `invalid_voice`            | Audio        | Requested voice is not available            | Choose a valid voice                                                                          |
| `insufficient_credits`     | All          | Not enough credits to complete the job      | [Add credits](https://magichour.ai/dashboard/my-plan)                                         |
| `invalid_parameters`       | All          | Invalid dimensions or other parameters      | Review parameters against the endpoint reference                                              |
| `processing_timeout`       | All          | Processing took too long and was terminated | Retry, or reduce the size/length of the input                                                 |

<Info>
  **No charge on failure**: When a job fails with an error, credits are refunded — you're never
  charged for a failed render.
</Info>

## Handling errors in code

- **Request errors** (`4xx`/`5xx`): inspect the HTTP status and the JSON `message`. Retry `5xx` responses with exponential backoff.
- **Job errors**: poll the project (or listen for the `*.errored` webhook) and branch on `error.code`. Treat unknown codes defensively.
